# 大数据系统与安全

# 一. 大数据安全的概念

  1. 大数据的概念
    1. 大数据的定义：大数据是指太大或太复杂，无法由传统数据处理软件处理的数据集。
    2. 大数据的生命周期包含：大数据的采集. 存储. 分析. 处理与共享五个阶段
    3. 大数据服务的核心四要素是：算法. 协议. 架构. 应用
    4. 大数据的5V 特征是：Volume（大量）. Velocity（高速）. Variety（多样）. Value（价值）. Veracity（准确）
    5. 大数据5V 特征中的客观特征是：Volume（大量）. Velocity（高速）. Variety（多样）
    6. 大数据5V 特征中的主观特征是：Value（价值）. Veracity（准确）
    7. 大数据5V 特征中的核心特征是：Value（价值）
    8. 2021 年全年我国数据产量达到6.6ZB
    9. 2020 年云上双11 平均每秒交易583000 笔
    10. 大数据数据类型多样化的显著特征是：数据半结构化或非结构化。
    11. “大数据”数据量巨大，价值密度却不见得高。
    12. 大数据数据质量问题来源于数据采集系统（例如，传感器网络）本身的质量问题，社交媒体上的谣言，电子商务网站上的水军等
    13. 大数据的根本目的：挖掘并创造价值
  2. 大数据的应用
    • 大数据的应用场景非常多，你能详细举个例子说明吗？
    点击查看
    1. 电商领域：相信大数据在电商领域的应用，大家已经屡见不鲜了，淘宝京东等电商平台利用大数据技术，对用户信息进行分析，从而为用户推送用户感兴趣的产品，从而刺激消费。
    2. 政府领域：“智慧城市”已经在多地尝试运营，通过大数据，政府部门得以感知社会的发展变化需求，从而更加科学化、精准化、合理化的为市民提供相应的公共服务以及资源配置。
    3. 医疗领域：医疗行业通过临床数据对比、实时统计分析、远程病人数据分析、就诊行为分析等，辅助一声进行临床决策，规范诊疗路径，提高一声的工作效率。
    4. 传媒领域：传媒相关企业通过收集各式各样的信息，进行分类筛选、清洗、深度加工，实现对读者和受众葛新华需求的准确定位和把握，并追踪用户的浏览习惯，不断进行信息优化。
    5. 安防领域：安防行业可实现视频图像模糊查询、快速检索、精准定位，并能够进一步挖掘海量视频监控数据背后的价值信息，反馈内涵知识辅助决策判断。
    6. 金融领域：用户画像的基础上，银行可以根据用户的年龄、资产规模、理财偏好等，对用户群进行精准定位，分析出潜在的金融服务需求。
    7. 电信领域：电信行业拥有庞大的数据，大数据技术可以应用于网络管理、客户关系管理、企业运营管理等，并且使数据对外商业化，实现单独盈利。
    8. 教育领域：通过大数据进行学习分析，能够为每位学生创设一个量身定做的个性化课程，为学生的多年学习提供一个富有挑战性而非逐渐厌倦的学习计划。
    9. 交通领域：大数据技术可以预测未来交通情况，为改善交通状况提供优化方案，有助于交通部门提高对道路交通的把控能力，防止和缓解交通拥堵，提供更加人性化的服务
  3. 大数据的安全
    1. 大数据安全的定义：针对大数据服务系统，从系统架构与认证授权. 计算与存储. 算法设计与数据采集等多个角度来分析其安全问题及解决方案。
    2. 大数据安全的目的：保证数据的机密性. 完整性和可用性
    3. 数据转移权指的是：用户可要求自己的个人数据畅通无阻地直接迁移至新的提供商，数据以机器可读的格式迁移。
    4. 被遗忘权指的是：每个数据主体有权要求数据控制者删除个人数据
    5. 算法公平性指的是：数据主体有权要求对算法自动决策给出解释
    6. 如果贷款申请人被自动决策拒绝时，有权寻求解释。这里的“权”指的是算法公平性。
    7. 根据《GB/T 22240-2020 信息安全技术网络安全等级保护定级指南》，对于定级对象，应确定其业务信息和系统服务安全受到破坏时所侵害的客体。
    8. “棱镜门”事件曝光后，美国政府声称“不窃听公民的数据，只是获取了公民通信的元数 据”。但实际上，元数据同样会暴露个人隐私。
    9. 元数据是：描述数据的数据。
    10. 元数据包含：数据的名称. 物理位置. 何时被何人访问等

# 二. 基础知识

  1. 密码算法
    1. 在密码设计中，最常用的两种技巧是代换（substitution）和置换（permutation）
    2. 代换密码substitution cipher)：建立一个代换表，加密时将需要加密的明文依次通过查表，替换为相应的字符，明文字符被逐个替换后，生成无任何意义的字符，即密文。这样的代换表，通常称为密钥。
    3. 置换密码(permutation cipher)，其特点是保持明文的所有字母不变，只是利用置换打乱了明文字母的位置和次序。
    4. 单表代换密码包括：位移密码. 仿射密码
    5. 维吉尼亚密码是一种多表代换密码。
    6. Shannon 在其1949 年的论文中介绍的一个新思想是通过“乘积”组合密码体制以提高密码安全性，所谓“乘积”指的是：如果密码体制不是幂等的，那么多次迭代有可能提高安全性
    7. 大数据中，许多数据都具有某种固有的模式。这主要是由数据冗余和数据结构引起的
    8. 分组密码的电码本模式（ECB）电码本模式直接利用分组密码对明文的各分组进行加密
    9. 公开密钥系统通常是复合式的，内含一个高效率的对称加密算法，用以加密信息，再以非对称加密算法加密对称密钥，以增进效率
    10. DES 加密算法使用的密钥为64 位，但有效密钥长度为56 位（另有8 位用于奇偶校验）
  2. 密码协议
    1. 密码协议指的是：两个或两个以上的参与者为完成密码通信中某项特定任务而约定的一系列有序的步骤。
    2. 密码协议的基本原则： i. 每个参与者必须了解协议，事先知道所要完成的所有步骤； ii. 每个（诚实）参与者都必须同意并遵守协议； iii. 协议必须是清晰的，即每一个步骤的意义必须明确，不会产生歧义； iv. 协议必须是完整的，即对于每一种可能发生的情况，都要有具体明确的操作步骤。
    3. Kerboros 协议是一种在线式（on-line）密钥分配协议
    4. Diffie—Hellman 密钥交换协议是一个典型的密钥协商协议
    5. 整数零知识证明：设和为两个大素数，= 。假设P 知道的因子，如果P 想让V 相信他知道的因子，并且P 不想让V 知道的因子，则P 和V 可以执行下面的协议： i. V 随机的选取一个大整数，计算≡( )，V 将告诉给P； ii. P 计算，使得≡( )，P 将告诉给V； iii. V 验证≡( )是否成立。
  3. 网络协议
    1. OSI 模型把网络通信的工作分为7 层，分别是物理层. 数据链路层. 网络层. 传输层. 会话层. 表示层和应用层。
    2. TCP/IP 协议中的应用协议包含：HTTP. SMTP. FTP. TELNET. SNMP
    3. TCP/IP 协议中的传输协议包含：TCP. UDP
    4. TCP 是面向连接的. 可靠的流协议。
    5. UDP 是不具有可靠性的数据报协议。
    6. 简述TCP 协议三次握手. 四次挥手过程
    7. IPSec 安全体系中，IKE 协议负责密钥管理，定义了通信实体间进行身份认证. 协商加密算法以及生成共享的会话密钥的方法
    8. SSL 协议位于应用层与传输层协议之间
    9. 简述Kerberos 协议过程

# 三. 大数据处理与存储及其安全隐私

  1. Hadoop

    • Hadoop 允许用户使用简单的编程模型实现跨机器集群对海量数据进行分布式计算处理
    • Hadoop 的核心三组件及其功能：
      1. Hadoop HDFS（分布式文件存储系统）：解决海量数据存储
      2. Hadoop YARN（集群资源管理和任务调度框架）：解决资源任务调度
      3. Hadoop MapReduce（分布式计算框架）：解决海量数据计算
    • Hadoop 之父是Doug Cutting
    • Hadoop 的实现源于Google 的学术论文
    • 处在Hadoop 生态圈的底层与核心地位的是HDFS
    • MapReduce 作为大数据生态圈第一代分布式计算引擎，由于自身设计的模型所产生的弊端，导致企业一线几乎不再直接使用MapReduce 进行编程处理，但是很多软件的底层依然在使用MapReduce 引擎来处理数据。
    • Hadoop 特性优点：扩容能力强. 成本低. 效率高. 可靠性强
    • Hadoop 集群包括两个集群：HDFS 集群. YARN 集群
    • HDFS 集群和YARN 集群两个集群逻辑上分离（两个集群互相之间没有依赖. 互不影响）. 通常物理上在一起（某些角色进程往往部署在同一台物理服务器上）
    • HDFS 集群和YARN 集群都是标准的主从架构集群
    • HDFS 集群
      1. 主角色：NameNode
      2. 从角色：DataNode
      3. 主角色辅助角色：SecondaryNameNode
    1. YARN 集群 i. 主角色：ResourceManager ii. 从角色：NodeManager

  2. HDFS

    1. 文件系统使用树形目录的抽象逻辑概念代替了硬盘等物理设备使用数据块的概念，用户不必关心数据底层存在硬盘哪里，只需要记住这个文件的所属目录和文件名即可.
    2. Windows 操作系统上的文件系统. Linux 上的文件系统. FTP 文件系统都是单机的文件系统
    3. 从根目录开始，节点路径具有唯一性
    4. 文件分布在不同机器上不利于寻找：元数据记录下文件及其存储位置信息，快速定位文件位置
    5. 大数据数据量大，单机存储遇到瓶颈：多机可横向扩展，机器不够加机器，理论上无限扩展
    6. 文件过大导致单机存不下. 上传下载效率低：文件分块存储在不同机器，针对块并行操作提高效率
    7. 硬件故障难以避免，数据易丢失：不同机器设置备份，冗余存储，保障数据安全
    8. HDFS 适合场景：大文件. 数据流式访问. 一次写入多次读取. 低成本部署. 廉价PC高容错
    9. HDFS 不适合场景：小文件. 数据交互式访问. 频繁任意修改. 低延迟处理
    10. HDFS 中的文件在物理上是分块存储（block）的，默认大小是128M，不足128M 则本身就是一块。
    11. NameNode 维护和管理文件系统元数据
    12. NameNode 是访问HDFS 的唯一入口
    13. DataNode 负责具体的数据块存储
    14. Secondary NameNode 充当NameNode 的辅助节点，但不能替代NameNode。
    15. NameNode 所在机器通常会配置有大量内存（RAM）。
    16. DataNode 所在机器通常配置有大量的硬盘空间。
    17. HDFS 在上传文件写数据过程中采用的一种数据传输方式为Pipeline 管道传输
    18. Pipeline 管道传输的意义：数据以管道的方式，顺序的沿着一个方向传输，这样能够充分利用每个机器的带宽，避免网络瓶颈和高延迟时的连接，最小化推送所有数据的延时
    19. 在HDFS pipeline 管道传输数据的过程中，传输的反方向会进行ACK 校验，确保数据传输安全。

  3. MapReduce

    1. MapReduce 的思想核心是“先分再合，分而治之”。
    2. Map 表示第一阶段，负责“拆分”：即把复杂的任务分解为若干个“简单的子任务”来并行处理。可以进行拆分的前提是这些小任务可以并行计算，彼此间几乎没有依赖关系。
    3. Reduce 表示第二阶段，负责“合并”：即对map 阶段的结果进行全局汇总。
    4. MapReduce 处理的数据类型是<key,value>键值对。
    5. MapReduce 局限性：
    6. 实时计算性能差：MapReduce 主要应用于离线作业，无法作到秒级或者是亚秒级得数据响应。
    7. 不能进行流式计算：流式计算特点是数据是源源不断得计算，并且数据是动态的；而MapReduce 作为一个离线计算框架，主要是针对静态数据集得，数据是不能动态变化得。
    8. 一个MapReduce 编程模型中只能包含一个Map 阶段和一个Reduce 阶段，或者只有 Map 阶段
    9. MapReduce 中的Shuffle 过程频繁涉及到数据在内存. 磁盘之间的多次往复

  4. Yarn

    1. FIFO Scheduler（先进先出调度器）即先提交的应用先运行。调度工作不考虑优先级和范围，适用于负载较低的小规模集群。

  5. Hadoop 安全

    1. 信息保障技术框架（IATF）将网络划分为了四个不同的焦点域：本地计算环境. 区域边界. 网络及基础设施. 支撑性基础设施
    2. 信息保障技术框架（IATF）的核心思想是纵深防御战略
    3. IaaS 是通过网络对外提供IT 基础设施资源，将计算. 存储等基础资源封装成服务，可帮助用户快速拥有硬件资源，面向企业或开发者
    4. 入侵检测系统（IDS）旁路部署在交换机上
    5. 入侵防护系统（IPS）串行部署在网络边界上
    6. 在Linux 中，SSH 的主要用途有：用户加密实现远程登录. 服务器之间的免密登录
    7. 防火墙五元组包括源IP 地址. 源端口. 目的IP 地址. 目的端口和传输层协议

# 四. 大数据算法及其安全

  1. 机器学习
    1. 机器学习三大核心要素：算法. 数据. 硬件
    2. 机器学习分类任务的输出为离散变量
    3. 机器学习回归任务的输出为连续变量
    4. 监督学习需要具有标签（label）的训练数据。
    5. 数据预处理时使用min-max 标准化. Z-分数标准化进行数据去量纲。
    6. 简述泛化能力和鲁棒性的差异
    7. 能画混淆矩阵. 能计算准确率（Accuracy）. 查准率（Precision）. 查全率（Recall）以及F1 分数
    8. 分数：> 1，查全率的权重高于查准率；< 1，查准率的权重高于查全率。
    9. AUC：ROC 曲线下的⾯积，取值范围⼀般在0.5 和1 之间。使⽤AUC 值作为评价标准是因为很多时候ROC 曲线并不能清晰的说明哪个分类器的效果更好，⽽作为⼀个数值，对应AUC 更⼤的分类器效果更好
    10. 使用2评价回归任务模型，2可能小于0，但不可能大于1.
    11. 为防止模型过拟合，提高模型的泛化能力，通常会在损失函数的后面添加一个正则化项。
    12. 使用梯度下降法训练机器学习模型时，当样本数据很多，计算量开销大，计算速度慢，可采用随机梯度下降法。
    13. 简述KNN 和KMeans 算法流程，并分析二者异同
    14. 机器学习是一种基于数据编程的人工智能方法
  2. 深度学习
    1. 深度学习是一种使用多层神经网络的机器学习方法
    2. 针对Sigmoid 作为激活函数时，两端易保和的缺点，可在激活函数前加入BN 层对数据进行归一化处理。
    3. 会出现“神经元坏死现象”的是ReLU 函数
    4. 利用( ,,)=( +) 简述神经网络中梯度是如何反向传播的
    5. 在计算机图像识别任务中，对图像中的每一个像素进行分类的任务被称作语义分割
    6. 输入为32×32× 3的数据经过大小为5×5× 3的卷积核（步长为1）处理后，输出数据大小为28×28× 1（(输入– 卷积核) / 步长+ 1）

# 五. 大数据共享及其安全隐私

  1. 差分隐私
    1. 在共享大数据时，采用匿名化处理并不能完全规避隐私泄露问题.
    2. 在共享大数据时，不公布某个个体的数据并不能完全规避该个体隐私泄露问题.
    3. 差分隐私含义：若分析者（Analyst）对包含Alice 的数据集D 和删除Alice 的数据集D‘（除此之外，两数据集一致）进行分析后，对于Alice 的信息掌握几乎是一样多的，那么我们就称数据集D 保护了Alice 的隐私。
    4. 差分隐私常见的实现方法：为输出数据添加噪声
    5. 拉普拉斯机制（Laplace Mechanism）受限于输出范围，并不总能兼顾数据的隐私性和可用性。
